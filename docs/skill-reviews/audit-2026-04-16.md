## Remaining recommendations

After this pass, the most common residual gaps across the fleet are:

1. **CV ❌ / ⚠️** — 14 skills cite calibration paths that don't resolve in this template
   repo (e.g. `prompts/completed/phase-58/`, `docs/adr/005-*.md`). Per
   `docs/skill-calibration-manifest.md`, downstream projects should populate
   per-skill `examples.md` once real artefacts exist.
2. **TR "do not use when" clauses** — 9 skills have no explicit exclusion clause. Each
   needs a one-line frontmatter or body addition.
3. **OA completion states** — several workflow skills (adr-approve, adr-new,
   ralph-pipeline, ralph-preflight, settings-hygiene) need their output templates to
   enumerate Success / Partial / Blocked / Already-done states with trigger conditions.
4. **Relationship map gaps** (CO ⚠️) — 16 skills are not listed in
   `docs/skill-design-standards.md §Relationship map`. A follow-up pass on that doc is
   needed.
5. **TE extractions** — several skills have >20-line inline blocks that should be moved
   to sibling `.md` files (ralph-prompt-create README template, ralph-pipeline
   `pipeline-deps.js` chain, ralph-prompt-auto Q1–Q12 block, corpus-query Python block,
   ralph-preflight report template).
